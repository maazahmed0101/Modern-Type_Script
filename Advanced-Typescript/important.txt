package.json

Ye project ki settings aur npm ki information rakhta hai.

Example:

{
  "name": "my-project",
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "dev": "node index.js"
  },
  "dependencies": {
    "typescript": "^5.9.0"
  }
}
tsconfig.json

Ye sirf TypeScript compiler ko batata hai ke code ko kaise compile karna hai.

Example:

{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "rootDir": "./src",
    "outDir": "./dist"
  }
}
Agar tumhara matlab ye hai:

"type": "commonjs"
2. module aur target change karne hain?

Ye tsconfig.json me hota hai:

    "module": "NodeNext",

  }

note important{
    Don Bhai😡, npx tsc ka matlab samajhna bahut important hai.

npx

npx ka kaam hota hai project ke andar install package ko run karna bina globally install kiye.

tsc

tsc ka matlab hai TypeScript Compiler.

Ye .ts (TypeScript) files ko .js (JavaScript) me convert karta hai.

Jab tum likhte ho:
npx tsc

To ye ye kaam karta hai:

Project me typescript package dhundta hai.
tsconfig.json ko read karta hai (agar maujood ho).
Usme jo settings hain unke mutabiq sari .ts files compile karta hai.
Output .js files ko outDir (jaise dist) me daal deta hai.
Example

Agar file hai:

// src/index.ts

let name: string = "Maaz";
console.log(name);

Aur tsconfig.json me ye ho:

{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./dist"
  }
}

Phir:

npx tsc

Run karne ke baad ye banega:

Project/
│
├── src/
│   └── index.ts
│
├── dist/
│   └── index.js
│
└── tsconfig.json

Aur dist/index.js me:

"use strict";
let name = "Maaz";
console.log(name);
}

Note{
    ager tum ek folder me ho or tum child folder ko target ker ke us ke file ko lena he to esa hoga
  {  node dist/index.js}
}
